# <a name="workflows-in-sharepoint"></a>Workflows in SharePoint

<p>Wenn SharePoint-Workflows keine e-Mails senden, hat Ihre Organisation möglicherweise die Exchange Online Absender Grenzwerte erkannt.&nbsp;</p> <p>Fehlermeldung "Workflow ist angehalten" kann auftreten, wenn Sie eines der folgenden Elemente haben:</p> <ul> <li>Sie verfügen über einen Workflow in SharePoint Online, der den SharePoint 2010 oder SharePoint 2013 Workflow-Plattformtyp verwendet.</li> <li>Der Workflow ist so konfiguriert, dass eine benutzerdefinierte e-Mail-Nachricht an mehr als 200 Benutzer gleichzeitig gesendet wird, mehr als 10.000 Empfänger pro Tag oder mehr als 30 Nachrichten pro Minute.</li> <li>Wenn Sie den Workflow ausführen, wird die e-Mail-Nachricht nicht gesendet, und Sie werden feststellen, dass die Fehlermeldung, der <strong>interne Status auf angehalten festgelegt</strong> oder an <strong>einen Empfänger nicht gesendet</strong> werden kann, angezeigt wird.</li> </ul> <p>Weitere Informationen finden Sie im folgenden <a href="https://support.office.com/en-us/article/-daily-email-limit-has-exceeded-and-your-workflow-has-been-suspended-or-unable-to-send-to-a-recipient-error-in-a-sharepoint-online-workflow-89d02169-5fa6-4259-affc-73edb6ca9fb6?ui=en-US&amp;rs=en-US&amp;ad=US">Artikel</a>.</p>
